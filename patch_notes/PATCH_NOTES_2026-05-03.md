# 패치 노트

날짜: 2026-05-03
프로젝트: LOL Team Builder

## 요약

이번 작업에서는 최근 전적 기반 분석 흐름과 팀 생성 보정 로직을 포함한 클라이언트 기능을 정리하고, 루트에 흩어져 있던 클라이언트 코드를 `client/` 디렉토리 아래로 재구성했습니다. 또한 실행 문서를 한글로 다시 정리하고, `dist/` 같은 빌드 산출물이 저장소에 올라가지 않도록 Git 설정을 정비했습니다.

## 상세 변경 사항

### 1. 팀 생성 계산 로직 보강

- 최근 경기 수를 바탕으로 신뢰도를 반영한 `최근 폼 보너스` 계산을 추가했습니다.
- 최근 포지션별 경기 비중과 승률을 바탕으로 `포지션 적합도 보너스` 계산을 추가했습니다.
- 팀 전투력 계산 시 티어 점수 외에 최근 승률, KDA, 포지션 적합도를 함께 반영하도록 조정했습니다.
- 라인별 전투력 차이, 최대 라인 차이, 원딜 맞라인 차이, 불균형 심각도를 함께 비교하도록 후보 선정 우선순위를 보강했습니다.
- 조건을 완전히 만족하는 조합이 없을 때 완화 규칙으로 다시 조합을 시도하는 fallback 흐름을 유지하도록 구성했습니다.

### 2. 최근 전적 분석 및 유저 프로필 생성 기능 정리

- Riot 계정 검색 기능을 클라이언트 흐름에서 사용할 수 있도록 정리했습니다.
- 최근 경기 목록, 선택 경기 상세, 포지션 통계, 챔피언 통계 UI 흐름을 유지 가능한 구조로 정리했습니다.
- 최근 승률, KDA, 평균 CS, 시야 점수, 딜량, 골드 등의 요약 데이터를 계산하는 Presenter 레이어를 구성했습니다.
- 선택한 계정을 유저 표에 추가할 때 최근 경기 기반 포지션과 메타 데이터를 함께 저장하도록 구성했습니다.
- 팀 결과 화면에 각 유저의 최근 폼과 포지션 적합도를 표시하도록 정리했습니다.

### 3. Riot 적재 도구 정리

- `client/tools/riot_loader.py`를 기준으로 수동 적재 도구를 클라이언트 패키지 내부로 이동했습니다.
- puuid 조회, 매치 ID 조회, 첫 경기 상세 조회, 최근 경기 DB 적재 흐름을 유지하면서 현재 구조에 맞게 정리했습니다.
- 저장된 계정을 검색하고 여러 계정을 선택해 일괄 적재하는 흐름을 계속 사용할 수 있도록 정리했습니다.

### 4. 클라이언트 디렉토리 구조 재편

- 다음 파일과 폴더를 루트에서 `client/` 아래로 이동했습니다.
  - `main.py` -> `client/main.py`
  - `main.spec` -> `client/main.spec`
  - `service/` -> `client/service/`
  - `ui/` -> `client/ui/`
  - `tools/` -> `client/tools/`
  - `styles/` -> `client/styles/`
  - `util/` -> `client/util/`
  - `data/` -> `client/data/`
- `client/__init__.py`를 추가해 패키지 경계를 명확히 했습니다.
- 스타일 로더가 새 클라이언트 루트를 기준으로 QSS 파일을 읽도록 수정했습니다.
- 설정 파일 로딩 경로가 `client/data/config.json`을 기준으로 동작하도록 수정했습니다.
- PyInstaller 스펙 파일이 `client/main.py`를 기준으로 빌드되도록 수정했습니다.

### 5. Git 추적 및 산출물 관리 정리

- `.gitignore`에서 `dist/` 전체를 무시하도록 변경했습니다.
- `client/team_builder_client_error.log`를 Git 추적 대상에서 제외했습니다.
- 클라이언트 데이터 JSON은 기본적으로 Git 추적 대상에서 제외되도록 조정했습니다.
- 기존에 추적 중이던 `dist/` 내부 파일은 저장소에서 제외되도록 정리했습니다.

### 6. 문서 갱신

- `README.md`를 현재 구조 기준으로 한글화했습니다.
- 클라이언트, Riot 로더, 서버, PyInstaller 실행 명령을 README에 정리했습니다.
- 패치노트를 `patch_notes/` 디렉토리에서 날짜별로 관리하도록 기준을 세웠습니다.

## 영향

- 클라이언트와 서버의 경계가 명확해져 이후 서버 EC2 배포와 로컬 클라이언트 개발을 분리해서 관리하기 쉬워졌습니다.
- 최근 전적 기반 데이터가 팀 생성 결과와 UI 표시 양쪽에 일관되게 반영됩니다.
- 빌드 산출물이 저장소에 섞이지 않아 커밋 관리가 더 깔끔해집니다.
